# 权限系统修复说明

## 问题描述

在管理员登录时出现错误：
```
获取管理员信息失败: Table 'volunteer_system.admin_permission' doesn't exist
```

这是因为数据库中缺少权限系统相关的表。

## 解决方案

### 步骤1：执行SQL脚本

使用MySQL客户端工具（如 Navicat、MySQL Workbench、或命令行）执行以下SQL文件：

**文件位置：** `database/fix_permission_system.sql`

#### 方法1：使用Navicat等图形化工具
1. 打开 Navicat，连接到 `volunteer_system` 数据库
2. 点击"查询" -> "新建查询"
3. 打开 `fix_permission_system.sql` 文件
4. 点击"运行"按钮执行

#### 方法2：使用MySQL命令行
```bash
mysql -u root -p volunteer_system < database/fix_permission_system.sql
```

### 步骤2：验证表创建

执行以下SQL语句检查表是否创建成功：

```sql
USE volunteer_system;

-- 检查表是否存在
SHOW TABLES LIKE '%permission%';

-- 应该看到：
-- permission
-- admin_permission

-- 查看权限数量
SELECT COUNT(*) FROM permission;
-- 应该显示 23 条权限记录

-- 查看权限列表
SELECT permission_key, permission_name FROM permission ORDER BY sort_order;
```

### 步骤3：重启后端服务

1. 停止后端服务（如果正在运行）
2. 重新启动后端服务
3. 尝试登录管理员账号

## 权限系统说明

### 创建的表

1. **permission 表**
   - 存储所有权限定义
   - 包含23个基础权限
   - 权限key使用连字符命名（如：`post-categories`）

2. **admin_permission 表**
   - 管理员与权限的关联表
   - 记录哪些管理员拥有哪些权限
   - 超级管理员无需配置，自动拥有所有权限

### 权限列表

| 权限Key | 权限名称 | 说明 |
|---------|---------|------|
| `dashboard` | 数据统计 | 所有管理员都可见 |
| `admins` | 管理员管理 | 仅超级管理员 |
| `volunteers` | 志愿者管理 | 可分配给普通管理员 |
| `activities` | 志愿活动 | 可分配给普通管理员 |
| `categories` | 活动分类 | 可分配给普通管理员 |
| `forum` | 论坛帖子 | 可分配给普通管理员 |
| `post-categories` | 帖子分类 | 可分配给普通管理员 |
| `banners` | 轮播图管理 | 可分配给普通管理员 |
| `comments` | 评论管理 | 可分配给普通管理员 |
| `announcements` | 系统公告 | 可分配给普通管理员 |
| `applications` | 报名申请 | 可分配给普通管理员 |
| `checkins` | 打卡记录 | 可分配给普通管理员 |
| `replenish` | 补录申请 | 可分配给普通管理员 |
| `goods` | 积分商品 | 可分配给普通管理员 |
| `orders` | 兑换订单 | 可分配给普通管理员 |
| `blockchain-query` | 区块链服务查询 | 可分配给普通管理员 |
| `blockchain` | 区块链证书 | 可分配给普通管理员 |
| `certificate-library` | 证书库管理 | 可分配给普通管理员 |

### 权限分配

#### 超级管理员（SUPER_ADMIN）
- 自动拥有所有权限
- 无需在 `admin_permission` 表中配置
- 可以为普通管理员分配权限

#### 普通管理员（ADMIN）
- 需要通过超级管理员分配权限
- 分配后需要重新登录才能生效
- 权限存储在 `admin_permission` 表中

### 特殊权限说明

#### 志愿者管理权限（volunteers）

对于 `volunteers` 权限，有特殊的权限分级：

| 操作 | 超级管理员 | 普通管理员（有volunteers权限） |
|------|-----------|---------------------------|
| 查看列表 | ✅ | ✅ |
| 添加志愿者 | ✅ | ✅ |
| 删除志愿者 | ✅ | ✅ |
| **修改志愿者** | ✅ | ❌ |

**说明：** 即使普通管理员拥有 `volunteers` 权限，也只能查看、添加、删除志愿者，不能修改志愿者的个人信息。只有超级管理员才能修改志愿者信息。

## 常见问题

### Q1: 执行SQL后仍然报错
**A:** 请检查：
- 数据库连接是否正确
- 是否在正确的数据库（volunteer_system）中执行
- 表是否真的创建成功（使用 `SHOW TABLES` 检查）
- 后端服务是否重启

### Q2: 普通管理员看不到任何菜单
**A:** 
- 确认该管理员已被分配权限
- 退出登录后重新登录
- 检查 `admin_permission` 表中是否有该管理员的权限记录
- 检查前端控制台是否有错误信息

### Q3: 如何为已存在的管理员分配权限？
**A:** 
1. 使用超级管理员登录
2. 进入"管理员管理"页面
3. 找到需要分配权限的管理员
4. 点击"编辑"或"权限配置"按钮
5. 勾选需要的权限
6. 保存后，该管理员需要重新登录

### Q4: 如何查看某个管理员的权限？
**A:** 
```sql
-- 查看管理员ID为2的权限
SELECT 
    a.username,
    p.permission_key,
    p.permission_name
FROM admin a
LEFT JOIN admin_permission ap ON a.id = ap.admin_id
LEFT JOIN permission p ON ap.permission_id = p.id
WHERE a.id = 2;
```

## 注意事项

1. ⚠️ 执行SQL脚本前建议备份数据库
2. ⚠️ 该脚本会删除已存在的 `permission` 和 `admin_permission` 表
3. ⚠️ 如果已有普通管理员配置了权限，执行脚本后需要重新分配
4. ⚠️ 超级管理员无需重新配置权限
5. ✅ 修改权限后需要重新登录才能生效
6. ✅ 前端会自动根据权限隐藏/显示菜单项
