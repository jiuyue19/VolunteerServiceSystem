# 权限系统修复说明（简化版）

## 问题描述

在管理员登录时出现错误：
```
获取管理员信息失败: Table 'volunteer_system.admin_permission' doesn't exist
```

## 原因分析

后端代码尝试从 `admin_permission` 表查询权限，但实际上系统使用的是 `admin` 表的 `permissions` 字段（JSON格式）来存储权限。

## 解决方案

### 方案说明

使用**简化的权限存储方案**：
- 权限定义存储在 `permission` 表（仅作为参考）
- 权限分配直接存储在 `admin.permissions` 字段（JSON数组格式）
- **不需要** `admin_permission` 关联表

### 修复步骤

#### 步骤1：执行SQL脚本

**文件位置：** `database/fix_permission_system_simple.sql`

**执行方法（任选一种）：**

##### 方法A：使用Navicat
1. 打开 Navicat，连接到 `volunteer_system` 数据库
2. 点击"查询" → "新建查询"

4. 点击"运行"按钮 ▶️

##### 方法B：使用MySQL命令行


## 权限存储格式

### admin表的permissions字段格式

JSON数组格式，例如：

```json
["volunteers", "activities", "forum", "applications"]
```

### 权限分配示例

#### 超级管理员
- `role` 字段为 `SUPER_ADMIN`
- `permissions` 字段可以为 `NULL` 或空（自动拥有所有权限）

#### 普通管理员示例1
```sql
UPDATE admin SET permissions = '["volunteers", "activities", "applications"]' WHERE id = 2;
```
该管理员拥有：志愿者管理、活动管理、报名申请权限

#### 普通管理员示例2
```sql
UPDATE admin SET permissions = '["forum", "comments", "banners"]' WHERE id = 3;
```
该管理员拥有：论坛管理、评论管理、轮播图管理权限

#### 清空权限
```sql
UPDATE admin SET permissions = NULL WHERE id = 4;
-- 或
UPDATE admin SET permissions = '[]' WHERE id = 4;
```

## 特殊权限说明

### 志愿者管理权限（volunteers）

对于 `volunteers` 权限，有特殊的权限分级：

| 操作 | 超级管理员 | 普通管理员（有volunteers权限） |
|------|-----------|---------------------------|
| 查看列表 | ✅ | ✅ |
| 添加志愿者 | ✅ | ✅ |
| 删除志愿者 | ✅ | ✅ |
| **修改志愿者** | ✅ | ❌ |

**说明：** 即使普通管理员拥有 `volunteers` 权限，也只能查看、添加、删除志愿者，不能修改志愿者的个人信息。只有超级管理员才能修改。

## 后端代码修改说明

### AdminController.java
- 超级管理员：自动获取所有权限并转换为JSON
- 普通管理员：直接返回 `admin.permissions` 字段的值

### VolunteerController.java
- 权限检查从 `admin.permissions` 字段读取（不再查询 `admin_permission` 表）

## 常见问题

### Q1: 执行SQL后仍然报错
**A:** 
1. 确认SQL执行成功（查看权限数量是否为23）
2. 重启后端服务
3. 清除浏览器缓存，重新登录

### Q2: 普通管理员看不到任何菜单
**A:** 
检查该管理员的 `permissions` 字段：
```sql
SELECT username, role, permissions FROM admin WHERE username = 'test_admin';
```
- 如果为 `NULL` 或 `[]`，需要分配权限
- 格式必须是有效的JSON数组

### Q3: 如何批量分配权限？
**A:** 
```sql
-- 为所有普通管理员分配基础权限
UPDATE admin 
SET permissions = '["volunteers", "activities", "applications"]' 
WHERE role = 'ADMIN';
```

### Q4: permissions字段应该用TEXT还是VARCHAR？
**A:** 
建议使用 `TEXT` 类型，因为权限较多时可能超过VARCHAR的长度限制。

```sql
-- 如果需要修改字段类型
ALTER TABLE admin MODIFY COLUMN permissions TEXT COMMENT '权限列表(JSON格式)';
```

## 注意事项

1. ✅ 权限存储在 `admin.permissions` 字段（JSON格式）
2. ✅ 超级管理员无需配置 `permissions` 字段
3. ✅ 修改权限后需要重新登录才能生效
4. ✅ `permission` 表仅作为权限定义参考
5. ⚠️ `permissions` 字段必须是有效的JSON数组格式
6. ⚠️ 权限key必须使用双引号（如：`"volunteers"`）

## 前端权限判断

前端会解析 `permissions` 字段：

```javascript
// 解析权限
let permissions = []
if (res.data.permissions) {
  try {
    permissions = JSON.parse(res.data.permissions)
  } catch (e) {
    permissions = []
  }
}

// 判断是否有权限
const hasPermission = (permission) => {
  if (isSuperAdmin.value) return true
  return userInfo.value.permissions.includes(permission)
}
```
