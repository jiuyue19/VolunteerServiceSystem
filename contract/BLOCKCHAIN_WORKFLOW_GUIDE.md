# 区块链上链工作流程说明

## 📋 功能概述

本系统实现了完整的区块链数据上链与证书生成工作流程，确保数据一致性和不可篡改性。

## 🔄 完整工作流程

### 步骤1：查询链上数据（可选但推荐）

**目的**：在上链前先查看当前链上数据，确认数据状态

1. 在"查询链上数据"区域选择志愿者
2. 点击"查询"按钮
3. 系统显示：
   - 钱包地址
   - **链上总时长**（当前区块链上的数据）
   - **链上补录时长**
   - **数据库总时长**（当前数据库中的数据）
   - **数据一致性**（链上数据与数据库数据是否一致）

**数据一致性说明**：
- ✅ **一致**：链上数据 = 数据库数据，表示数据已同步
- ❌ **不一致**：链上数据 ≠ 数据库数据，需要同步

---

### 步骤2：数据上链（含确认机制）

**目的**：将数据库中的最新数据同步到区块链

#### 2.1 发起上链请求

1. 在"数据上链"区域选择志愿者
2. 点击"同步上链"按钮

#### 2.2 数据确认对话框 ⚠️ **关键步骤**

系统自动弹出**上链数据确认对话框**，显示：

| 字段 | 说明 |
|------|------|
| 志愿者姓名 | 即将上链的志愿者 |
| 钱包地址 | 区块链钱包地址 |
| **当前链上总时长** | 当前区块链上已有的数据 |
| **数据库总时长** | 数据库中的最新数据 |
| **即将同步的时长** | 将要上链的数据（= 数据库总时长）|
| 数据状态 | 已同步 / 需要同步 |

**⚠️ 重要提示**：
```
数据上链后将无法修改，请确保数据准确无误！
```

#### 2.3 确认操作

- **取消上链**：关闭对话框，不执行上链操作
- **确认上链**：执行上链操作，将数据库数据同步到区块链

#### 2.4 上链成功

- 显示交易哈希（Transaction Hash）
- **自动查询最新链上数据**，确保数据一致性
- 更新数据一致性状态

---

### 步骤3：生成区块链证书

**目的**：基于链上数据生成志愿服务证书

#### 3.1 前置条件

- 必须先执行"查询链上数据"
- 钱包地址必须有效
- 链上必须有数据

#### 3.2 打开证书生成对话框

1. 在查询结果下方点击"为该志愿者生成证书"
2. 系统自动填充链上数据：
   - **志愿者姓名**（自动填充，禁止修改）
   - **服务时长**（从链上读取，禁止修改）
   - **补录时长**（从链上读取，禁止修改）
   - **证书备注**（可选填写）

#### 3.3 数据来源说明 ℹ️

```
证书数据来源于区块链
以下数据自动从区块链读取，不可手动修改
```

所有核心数据（姓名、时长）均来自**区块链**，确保：
- ✅ 数据真实性（区块链认证）
- ✅ 数据不可篡改性
- ✅ 证书与链上数据完全一致

#### 3.4 生成证书

1. 点击"生成证书"按钮
2. 系统自动下载证书PNG图片
3. 证书包含：
   - 志愿者姓名
   - 累计服务时长
   - 补录时长
   - 区块链认证标识
   - 钱包地址
   - 证书编号
   - 生成时间

## 🔒 数据一致性保证

### 1. 上链前确认

- 显示链上当前数据 vs 即将上链的数据
- 用户可以对比确认，避免错误上链

### 2. 上链后自动验证

- 上链成功后立即查询最新链上数据
- 确保数据已正确同步到区块链

### 3. 证书数据不可修改

- 证书数据100%来自区块链
- 字段设为 `disabled`，不允许手动修改
- 确保证书真实性

## ⚠️ 注意事项

### 1. 数据上链前必须确认

- 上链的数据**无法修改**
- 请仔细核对确认对话框中的数据
- 确保数据库数据准确无误后再上链

### 2. 证书数据来源

- 证书数据**100%来自区块链**
- 不允许手动输入或修改核心数据
- 这确保了证书的真实性和可信度

### 3. 数据一致性

- 上链成功后会自动查询验证
- 如果数据不一致，检查：
  - Geth 节点是否正常运行
  - 交易是否被确认
  - 网络连接是否正常

### 4. 钱包地址绑定

- 志愿者必须先绑定钱包地址才能上链
- 钱包地址格式：`0x` 开头，42个字符
- 示例：`0x1be31a94361a391bbafb2a4ccd704f57dc04d4bb`

## 🎯 最佳实践

### 1. 推荐工作流程

```
查询链上数据 → 确认数据一致性 → 上链（如需要）→ 验证 → 生成证书
```

### 2. 数据同步建议

- **定期查询**：定期查询链上数据，了解同步状态
- **批量上链**：积累一定数据后统一上链，节省Gas费用
- **及时验证**：上链后立即验证数据一致性

### 3. 证书生成建议

- 只为已上链的志愿者生成证书
- 确保链上数据最新（刚上链完成）
- 保存证书文件备份

## 🔧 故障排查

### 问题1：数据不一致

**原因**：链上数据与数据库数据不同步

**解决方案**：
1. 点击"同步上链"
2. 确认数据后上链
3. 系统自动验证数据一致性

### 问题2：无法生成证书

**原因**：未查询链上数据或钱包地址无效

**解决方案**：
1. 先执行"查询链上数据"
2. 确保钱包地址有效
3. 确保链上有数据

### 问题3：证书数据为空

**原因**：志愿者未上链或链上数据为0

**解决方案**：
1. 确认志愿者已上链
2. 查询链上数据确认
3. 如链上数据为0，重新上链
